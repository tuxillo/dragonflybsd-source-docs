    IPC & Sockets Overview¶

    DragonFly BSD provides multiple inter-process communication mechanisms, from low-level kernel primitives to POSIX-standard APIs. This section documents the kernel implementation of these facilities.

    Socket Layer¶

    The socket abstraction provides a uniform interface for network and local communication:

    • Mbufs - Network memory buffer management
    • Sockets - Socket API implementation and socket buffers
    • Unix Domain Sockets - Local IPC via filesystem namespace
    • Protocol Dispatch - Protocol family registration and netisr

    Pipes¶

    Traditional Unix pipes for unidirectional byte streams between related processes:

    • Pipes - pipe() system call, direct data transfer optimization

    Message Queues¶

    Two message queue implementations:

    • POSIX Message Queues - mq_open(), priority-based queuing
    • System V Message Queues - msgget(), msgctl(), msgsnd(), msgrcv()

    System V IPC¶

    Classic System V IPC primitives:

    • System V Message Queues - Message passing
    • System V Semaphores - Counting semaphores with undo support
    • System V Shared Memory - Shared memory segments

    All System V IPC facilities share common permission checking via ipcperm() and use integer keys for namespace management.

    Comparison¶

    Mechanism Scope Data Model Persistence Best For
    Pipes Related processes Byte stream Process lifetime Simple parent-child
    Unix Sockets Local system Stream/Datagram Filesystem Local services, fd passing
    POSIX Mqueue System-wide Messages Kernel (can persist) Priority messaging
    SysV Mqueue System-wide Messages Kernel Legacy compatibility
    SysV Semaphores System-wide Counters Kernel Process synchronization
    SysV Shm System-wide Raw memory Kernel High-bandwidth sharing
    TCP/UDP Sockets Network Stream/Datagram Connection lifetime Network communication
